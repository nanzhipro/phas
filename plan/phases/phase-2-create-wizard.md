# Phase 2: Implement VM Creation Wizard and Admission Gates

## 阶段定位

在 phase-1 已经具备稳定领域模型和 bundle 持久化基础后，本阶段负责把“创建一台 VM”的产品入口真正落到 GUI 上：用户能输入名称、选择 ISO、挑选资源预设，并在创建前看到明确的阻断原因或软警告。

## 必带上下文

- plan/common.md
- Phase 1 已完成
- PRD.md 中的 6.3、7.1、8.1、8.1.1、8.1.2、8.1.3、15.2、15.3、15.4

## 阶段目标

- 交付可从首页进入的原生 macOS 创建向导，收集 VM 名称、ISO 路径和资源配置。
- 展示最小必要的宿主机信息，包括总内存、CPU 建议和推荐预设。
- 在真正落盘前执行一次创建前准入检查，并把结果分为硬性阻断与软性警告。
- 基于 phase-1 的 bundle store 完成 Draft VM 的真实创建，让首页能看到唯一 VM 的摘要。

## 实施范围

- 首页与创建向导 UI。
- 创建请求模型、预设模型、宿主机能力快照与 admission validator。
- 单 VM 创建用例：通过校验后生成 Draft 记录并落盘 bundle。
- 与本阶段行为直接相关的单元测试和使用文档。

## 本阶段产出

- 可操作的创建向导与首页入口。
- 一套准入校验逻辑，覆盖宿主架构、macOS 版本、ISO 可读性、显式架构错配、单 VM 限制、资源下限与安全阈值、磁盘空间检查。
- 支持“继续但带警告”的镜像支持级别提示机制。
- 创建完成后可在首页看到 VM 摘要，且重复创建会被产品边界阻断。

## 明确不做

- 不实现真实 VM 启动、EFI 装配、VZVirtualMachineConfiguration 或运行窗口。
- 不实现删除、启动、停止、重试等运行期动作。
- 不实现多 VM 列表管理、复杂导航、系统诊断面板或实时资源监控。

## 完成判定

- 首页存在可触发的“Create Virtual Machine”入口，能弹出向导并完成一次 Draft VM 创建。
- 创建前准入校验能区分 blocking/warning 两类结果；blocking 会禁止创建，warning 可继续但必须可见。
- 成功创建后，bundle 真实写入 phase-1 约定路径，首页能显示已创建 VM 的名称、状态和资源摘要。
- 自动化测试覆盖资源阈值、单 VM 限制、ISO 可读性/显式架构错配判定，以及创建用例会调用 bundle bootstrap。
