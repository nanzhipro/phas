# phas 通用规划约束

本文件是 phas 全部 Phase 的长期稳定约束来源。任何单步执行都必须把本文件作为完整上下文的一部分，而不是只看局部任务。

## 结论

这是一个面向 Apple silicon macOS 14+ 的原生本地 Linux VM MVP 项目，目标是在单仓库内交付单虚拟机创建、图形化安装、持久化启动、基础联网、错误恢复和最小可诊断能力的完整闭环，而不是做通用虚拟化平台。

本规划将以下要求视为硬约束，而不是“后续优化项”：

- 严格以 PRD.md 为产品边界，只交付单 VM MVP，不提前实现多 VM、快照、桥接网络、共享目录、剪贴板同步、USB、音频、Rosetta 或 Intel 支持。
- 运行底座只能使用 Apple 原生 Virtualization.framework 及其配套系统框架，不引入 QEMU、UTM、Parallels、VMware 或其他第三方虚拟化运行时。
- 宿主机支持范围锁定为 Apple silicon 与 macOS 14+；主验收镜像锁定为 Ubuntu Desktop ARM64，Fedora Workstation ARM64 仅做补充验证，不扩展到更多发行版承诺。
- 产品必须使用固定 VM bundle 路径 `~/Library/Application Support/phas/VMs/<vm-id>.vmbundle/`，并在产品层配置中维护稳定 VM ID、schema version、启动源与网络模式字段。
- 任何错误都必须映射到明确的用户可见恢复动作；除用户显式删除外，不允许以“自动修复”为名静默破坏 VM 数据或重置状态。
- 每个实施 Phase 都必须做最窄但真实的可执行验证；能跑构建、测试、应用启动、脚本检查时，不允许只靠静态阅读或 diff 自证完成。
- 日志与诊断仅保存在本机，且禁止记录 guest 屏幕内容、键盘输入内容或 guest 内部文件内容。
- README 只保留高层概览、运行入口和关键限制；详细设计、验证矩阵和执行细节放在 plan 文档或独立 docs 页面，避免 README 漫灌。
- 如果使用项目生成器或自动生成 Xcode 工程，产物必须与当前本地 Xcode 工具链兼容，并通过真实构建命令验证；不能留下只在更新版工具链可打开的工程格式。

## 产品/项目目标

交付一个可在 macOS 本地反复创建、安装、启动、停止和删除单台 Linux VM 的原生 GUI 应用，并用最少功能完成产品级闭环验证。

## 非目标

不把本项目扩展成多机管理平台、企业级虚拟化编排器或跨架构仿真工具。

- 不做多台 VM 同时管理、快照、克隆、回滚、桥接网络和自定义网络拓扑。
- 不做共享目录、剪贴板同步、USB 透传、音频输入输出、Rosetta for Linux、Intel Mac 首版支持。
- 不做 Linux 镜像自动下载、无人值守安装、现有第三方磁盘格式导入。

## 硬性工程约束

### 平台与工具链

- 目标应用为 macOS 14+ 原生 GUI 应用，宿主机范围锁定 Apple silicon。
- 核心实现语言与宿主框架以 Swift、SwiftUI/AppKit、Foundation、OSLog、Virtualization.framework 为主。
- 构建与验证必须能被当前本地 Xcode 工具链实际执行；若引入 project generator，必须验证生成产物仍可被当前工具链打开与构建。
- 面向运行时的验证优先使用 `xcodebuild`、XCTest 或可重复脚本，不把“理论上可运行”当作验收结果。

### 依赖边界

禁止引入以下内容：

- 任何第三方虚拟化底座、桥接网络 entitlement 依赖、或绕过 Apple 原生沙箱/entitlement 约束的方案。
- 与 MVP 无关的重型 UI 框架、跨平台壳层、远程同步基础设施或需要云服务才能完成主流程的依赖。

允许使用的仅限：

- Apple 原生系统框架与本地测试框架。
- 为提高开发效率所必需且不改变运行底座的轻量工具，但必须有明确退出成本且不破坏当前工具链兼容性。

## 质量底线

- 每个 Phase 完成时都要留下可复跑的验证证据，至少包括构建命令、测试命令、脚本检查或明确的手动验证清单。
- 运行相关功能必须优先验证真实状态转移、错误恢复与数据持久化，而不是只验证 happy path 的局部函数。
- 删除逻辑、持久化逻辑和状态恢复逻辑必须把“避免误删宿主机其他文件”和“避免状态误判”作为最高优先级。
- entitlement、日志、状态机和 VM bundle 清理必须可审计，不能隐藏在不可追踪的全局副作用里。

## 其他不可跨越的边界

- 对外产品承诺以 Ubuntu Desktop ARM64 完整闭环为准；Fedora Workstation ARM64 只作为补充验证，不得反向扩大首发承诺。
- 首版按开发者签名后的直接分发设计，不以 Mac App Store 审核约束反向定义架构。
- 当产品无法可靠判断 VM 一致性时，优先进入 Error 并要求显式重试；不以猜测状态替代数据安全。
