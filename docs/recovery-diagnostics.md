# Recovery and Diagnostics

phase-5 为单 VM MVP 补上“应用重启后仍然可信”的恢复规则，并把它变成用户可见能力。

## Relaunch 语义

- 应用启动时会重新读取固定 bundle 路径中的 VM 记录
- 如果记录状态是 `Draft` 或 `Stopped`，产品认为该状态稳定，可以继续展示并允许恢复运行窗口入口
- 如果记录状态是 `Running` 或 `Installing`，但当前并不存在真实的运行会话，产品不会伪装成“仍在运行”
- 这类瞬时态会被显式映射为 `Error`，并要求用户查看日志后决定是重试启动还是回收到 `Stopped`

## 诊断面板

首页与运行窗口侧栏都会展示同一套恢复结论：

- 当前恢复结论标题与严重级别
- 最近错误/恢复摘要
- bundle 路径与日志路径
- 可执行恢复动作

## 当前恢复动作

- `Retry Start`：沿用 phase-4 的启动路径重新创建或复用运行会话
- `Recover to Stopped`：把当前记录显式回收为安全的 `Stopped` 状态，不做静默数据删除
- `Reload State`：重新从 bundle 读取当前记录并重新评估恢复结论

## 当前边界

- 不做静默自动修复，更不会删除或重置 bundle 数据
- 不做远程日志上报或诊断导出
- 运行窗口是否在 relaunch 后恢复，只在状态稳定时进行；脏状态不会自动重开运行窗口
- phase-6 再补验证矩阵自动化，phase-7 再补 release 文档和验收台账
